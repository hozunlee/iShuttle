# iShuttle — 업무 프로세스 (WORKFLOW)

> work/ 폴더는 백엔드·프론트엔드가 소통하는 유일한 허브다
> 구두 협의 금지. 모든 결정은 문서로 남긴다.

---

## 폴더 구조

```
work/
├── WORKFLOW.md       ← 이 파일. 프로세스 규칙
├── API_CHANGES.md    ← API 스펙 변경 이력 (계약 변경 시 필수)
├── DONE.md           ← 완료된 작업 로그 (요약)
├── tasks/            ← 진행 중 / 예정 태스크
│   ├── BE_003_TrackNetV3_통합.md
│   └── FE_006_업로드_스냅샷_팀지정.md
└── done/             ← 완료된 태스크 파일 보관
    ├── BE_001_백엔드_스캐폴딩.md
    └── FE_001_프론트엔드_스캐폴딩.md
```

---

## 태스크 상태 흐름

```
[TODO] → [IN_PROGRESS] → [BLOCKED] → [REVIEW] → [DONE]
                              ↑
                         상대방 작업 대기 시
```

---

## 태스크 파일 네이밍 규칙

```
{담당}_{번호}_{작업명}.md

BE_001_코트감지_모듈.md       백엔드 1번 태스크
FE_001_업로드화면.md          프론트 1번 태스크
SHARED_001_API스펙_확정.md    공동 결정 필요
```

---

## 태스크 파일 템플릿

```markdown
# [BE/FE_번호] 작업명

> 상태: TODO | IN_PROGRESS | BLOCKED | REVIEW | DONE
> 담당: BE | FE
> 생성: YYYY-MM-DD
> 완료: YYYY-MM-DD (완료 시)

## 목표

한 줄로: 이 작업이 끝나면 무엇이 동작하는가

## 완료 기준 (Done Criteria)

- [ ] 구체적으로 검증 가능한 항목 1
- [ ] 구체적으로 검증 가능한 항목 2

## 입력 (Input)

이 작업에 필요한 선행 조건 또는 데이터

## 출력 (Output)

이 작업이 만들어내는 결과물 (파일, API, 컴포넌트 등)

## 의존성

- 선행 필요: BE_001 완료 후 가능
- 후행 영향: FE_002가 이 결과를 사용

## 메모 / 이슈

작업 중 발생한 문제, 결정 사항, 참고 링크
```

## API 변경 프로세스

> API 스펙(CONTEXT.md의 API 계약 섹션)이 바뀌면 아래 절차를 따른다

1. 변경 제안자가 `work/tasks/SHARED_번호_API변경_내용.md` 생성
2. 상대 파트 확인 후 `work/API_CHANGES.md`에 이력 추가
3. `CONTEXT.md` API 계약 섹션 업데이트
4. 양쪽 모두 변경 반영 후 태스크 DONE 처리

## 태스크 완료 처리 규칙

1. 태스크 파일 상태를 `DONE` 으로 변경
2. `work/tasks/` → `work/done/` 으로 파일 이동
3. `work/DONE.md` 에 요약 항목 추가

## 규칙 요약

| 규칙                  | 내용                                            |
| --------------------- | ----------------------------------------------- |
| 문서 우선             | 구두 합의 없음. 결정은 반드시 파일로            |
| API 계약 불변         | 변경 시 SHARED 태스크 생성 후 양쪽 합의         |
| 완료 기준 필수        | 태스크에 Done Criteria 없으면 착수 금지         |
| 블로커 즉시 공유      | 막히면 태스크 상태를 BLOCKED로 바꾸고 이유 기재 |
| output/ 건드리지 않음 | 런타임 결과물. 수동 수정 금지                   |
| 완료 파일 이동        | DONE 상태 태스크는 tasks/ → done/ 으로 이동     |
