## 목표

파이프라인이 완료된 후 `/results/{jobId}` 화면이 랠리 0개·정상 데이터 모두에서 깨지지 않고 렌더링된다.

## 문제 분석

수동 테스트에서 파이프라인은 완료됐으나 랠리 0개 상태로 결과 화면으로 이동함.
결과 화면(Results.tsx)이 빈 배열을 처리하는지 확인 필요.

추가로 발견된 폴링 과다 문제:
```
GET /api/status ... (x21 이상, /api/results 호출 이후에도 계속)
```
`handleDone()` 이 WS 콜백에서 호출될 때 `clearInterval(pollInterval)` 이 누락됨.

## 완료 기준 (Done Criteria)

- [ ] 랠리 0개일 때 Results 화면이 "랠리 데이터가 없습니다" 메시지 표시 (크래시 없음)
- [ ] Results 화면에서 히트맵 이미지 렌더링 확인
- [ ] Results 화면에서 AI 리포트 텍스트 렌더링 확인
- [ ] `/api/results` 성공 후 `/api/status` 폴링이 즉시 중단됨

## 구현

### 폴링 중단 버그 수정 (`Analyzing.tsx`)

```typescript
const handleDone = async () => {
  if (finished) return;
  finished = true;
  clearInterval(pollInterval);  // ← 추가
  try {
    const result = await getResults(jobId);
    setDone(result);
    navigate(`/results/${jobId}`);
  } catch (e) {
    setError("결과를 불러오지 못했습니다");
  }
};
```

### Results.tsx 빈 배열 처리

```tsx
{rallies.length === 0 ? (
  <div className="text-gray-400 text-center py-12">
    랠리 데이터가 없습니다. 영상을 다시 업로드해주세요.
  </div>
) : (
  rallies.map(r => <RallyCard key={r.id} rally={r} />)
)}
```
